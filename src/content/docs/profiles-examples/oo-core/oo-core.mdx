## Shared Layer Versus Church Profile

The shared layer should contain concepts that retain the same meaning across participating churches. A church profile owns its rite-specific vocabulary, language rules, calendars, music, sacred-space model, and approval process.

| Put in the shared layer                                   | Put in a church profile                   |
| --------------------------------------------------------- | ----------------------------------------- |
| Stable IDs and references                                 | Native role titles                        |
| Localized text containers                                 | Calendar calculations and precedence      |
| Generic utterance, reading, rubric, and block composition | Chant systems and mode vocabulary         |
| Source, provenance, and authority structures              | Sanctuary zones and access rules          |
| Extension namespace mechanism                             | Jurisdiction-specific review requirements |

Similarity is not enough for standardization. When two traditions use related terms with different liturgical consequences, preserve both in their profiles and document the relationship rather than collapsing them.
